= Testing a decision service

// Purpose statement for the assembly
As a business analyst or business rules developer, you can use test scenarios in {CENTRAL} to test a decision service before the project is deployed, or use API calls in the {KIE_SERVER} to test a decision service after the project is deployed. Testing a decision service ensures that all project assets and settings function properly and as expected.

.Prerequisites
* The team and project for the decision service have been set up in {CENTRAL}. For details, see {URL_GETTING_STARTED_DECISION_SERVICE}[_{GETTING_STARTED_DECISION_SERVICE}_].
* All business rules and other assets have been defined and validated for the decision service. For example, see {URL_GUIDED_DECISION_TABLES}[_{GUIDED_DECISION_TABLES}_].
