// Purpose statement for the assembly
As a business rules developer or systems administrator, you can use KIE APIs to interact with {KIE_SERVERS}, KIE containers, and business assets in {PRODUCT}. You can use the {KIE_SERVER} REST API and Java client API to interact with KIE containers and business assets (such as business rules, processes, and solvers), the {CONTROLLER} REST API and Java client API to interact with {KIE_SERVER} templates and instances, and the Knowledge Store REST API to interact with spaces and projects in {CENTRAL}.

.Prerequisites
* {PRODUCT} is installed and running. For installation and startup options, see {URL_PLANNING_INSTALL}[_{PLANNING_INSTALL}_].
* You have access to {PRODUCT} with the following user roles:
** `kie-server`: For access to {KIE_SERVER} API capabilities, and access to {HEADLESS_CONTROLLER} API capabilities without {CENTRAL} (if applicable)
** `rest-all`: For access to {CENTRAL} API capabilities for the built-in {CONTROLLER} and for the {CENTRAL} Knowledge Store
** `admin`: For full administrative access to {PRODUCT}
+
Although these user roles are not all required for every KIE API, consider acquiring all of them to ensure that you can access any KIE API without disruption. For more information about user roles, see {URL_PLANNING_INSTALL}[_{PLANNING_INSTALL}_].
